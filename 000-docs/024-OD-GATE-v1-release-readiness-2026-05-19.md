## Conditions on the GO verdict

Two findings surfaced during the gate run that don't block release
but must be addressed in the v1.0 cut (B12):

### Condition 1: `ico --version` reports the wrong version

`packages/kernel/src/version.ts` hard-codes `'0.1.0'`. The CLI's
`buildProgram()` imports `version` from `@ico/kernel` for its
`.version()` call. Result: a fresh `ico --version` invocation on a
v0.22.0 install prints `0.1.0`.

**Fix during B12:** the release script must either bump
`packages/kernel/src/version.ts` in lock-step with the root
`package.json` and `version.txt`, or change `buildProgram()` to
read from the CLI's own package.json. Recommend the latter — single
source of truth for what the operator sees.

### Condition 2: Coverage shortfall is documented but not closed

Per item 3 above. Not blocking, but a post-v1.0 coverage-uplift
bead should land in the next planning cycle to prevent further
drift.
